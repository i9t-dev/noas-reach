This is a CiviCRM extension project. Follow these guidelines when working in this repository.

## Project Context

- **Type**: CiviCRM extension
- **Extension key**: `noas_reach` (short name) / `noas-reach` (long name)
- **Class prefix**: `CRM_NoasReach`
- **Primary language**: PHP 7.4+
- **Framework**: CiviCRM

## Codebase Structure

- `ext/noas-reach/` - Extension source directory
  - `noas_reach.civix.php` - Auto-generated civix boilerplate (DO NOT edit manually)
  - `info.xml` - Extension metadata
- `core/` - CiviCRM core (if present)

## Rules for Mistral Vibe

### Editing

- **DO NOT edit** `ext/noas-reach/noas_reach.civix.php` directly — it is auto-generated by civix and will be overwritten
- Use civix command-line tool to add hooks, pages, or other extension components
- For custom code, create files in the extension directory following CiviCRM naming conventions
- Match existing PHP style: PSR-12 with 2-space indentation

### PHP Standards

- Use `CRM_NoasReach_` prefix for all custom classes
- Use `E::` shortcut for `CRM_NoasReach_ExtensionUtil` (already defined in civix.php)
- Always include type hints where possible (PHP 7.4+)
- Use `ts()` for translatable strings with the extension domain

### CiviCRM Patterns

- Hook implementations should follow CiviCRM naming: `_noas_reach_` prefix
- Navigation menu items use `hook_civicrm_navigationMenu`
- Resource URLs use `E::url()` and paths use `E::path()`
- Translations use `E::ts()` with domain `noas-reach`

### Tools

- civix: Used to generate extension boilerplate
- Run `civix generate:module` to create new components
- Run `civix upgrade` to regenerate civix.php after changes

### Testing

- If test files exist, use PHPUnit
- CiviCRM provides `CRM_Utils_Array` and other utility classes for testing
- Mock CiviCRM core functions when testing in isolation

### Git

- Commit messages: Use imperative mood, 50 char subject line
- Include `Generated by Mistral Vibe.` and co-authorship when committing
- Do not push directly to `main` without review

## When in Doubt

- Check CiviCRM documentation at https://docs.civicrm.org/
- Look at existing CiviCRM extensions for patterns
- civix-generated code follows CiviCRM best practices
